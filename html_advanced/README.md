# 🌐 HTML - Structure d'une page Web (Techium)

Bienvenue dans le projet de structuration HTML pour **Techium** ! 

L'objectif principal de ce projet est de maîtriser les bases fondamentales du développement web : **les balises HTML**. Dans ce projet, il n'y a **aucun CSS ni aucun style**. Les pages finales seront "moches" et non stylisées, et c'est tout à fait normal ! Toute l'attention est portée sur l'exactitude sémantique, la propreté de la structure et le respect des détails (minuscules vs majuscules, syntaxe exacte, etc.).

## 🎯 Objectifs d'apprentissage

À la fin de ce projet, tu dois être capable d'expliquer ces concepts à n'importe qui, sans l'aide de Google :

### 1. Bases et règles du HTML
* Quelles directives spécifiques suivre pour écrire un code HTML propre.
* Comment créer correctement le squelette de base d'une page HTML5.
* Comment structurer correctement l'ensemble d'une page HTML de A à Z.

### 2. Sémantique et Structure
* Comment utiliser les balises sémantiques HTML pour donner du sens à une page web.
* Les différences précises et les cas d'utilisation entre une balise `<div>` (bloc) et `<span>` (en ligne).
* La valeur sémantique et le rôle exact des balises : `<header>`, `<main>`, `<footer>`, `<article>`, `<nav>`, `<section>` et `<aside>`.
* Comment utiliser les titres (`<h1>` à `<h6>`) et pourquoi il est crucial de respecter l'ordre hiérarchique pour le référencement (SEO) et l'accessibilité.

### 3. Contenu et Éléments Média
* Comment créer des listes ordonnées (`<ol>`), non ordonnées (`<ul>`) et de description (`<dl>`).
* Les différences, avantages et inconvénients des formats de médias : **SVG, GIF, PNG, JPG**.
* Comment structurer des données sous forme de tableau à l'aide de la balise `<table>`.
* Comment intégrer un élément vidéo natif (`<video>`).
* Comment intégrer un fichier audio (`<audio>`).
* Comment intégrer du contenu externe de manière sécurisée (par exemple, avec un `<iframe>`).

## 📋 Exigences et Contraintes

* **Documentation :** Un fichier `README.md` à la racine du dossier du projet est obligatoire.
* **Validation :** Tout le code doit être conforme aux normes du W3C et valider le test du [W3C-Validator](https://validator.w3.org/).
* **Nom de l'entreprise :** **Techium** est le nom officiel de l'entreprise qui doit être utilisé sur toutes les pages web créées.
* **Rigueur :** Les détails sont importants ! Fais très attention aux majuscules/minuscules dans les balises et évite les fautes de frappe.
